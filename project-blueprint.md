# 塔罗个人品牌与教学知识库项目蓝图

## 定位

这是一个独立于现有个人网站的新系统。它不是单纯的展示站，而是你的教学资产库、学生资料入口和未来 AI 助教的基础。

## 第一阶段目标

- 建立一个可搜索的教学知识库
- 区分公开、团课、一对一、仅自己四种内容
- 把幕布资料逐步迁移进来
- 让你备课、找案例、给学生补资料的成本明显下降

## 网站结构

1. 总览
   - 当前内容状态
   - 最近整理的资料
   - 常用入口

2. 学生入口
   - 公开学习
   - 团课学生
   - 一对一学生
   - 学生作业/反馈入口

3. 知识库
   - 大阿尔卡那
   - 小阿尔卡那
   - 牌阵与提问
   - 主题专题
   - 学生常见误区

4. 案例库
   - 感情关系
   - 事业选择
   - 自我成长
   - 复杂关系
   - 教学示范案例

5. 私人后台
   - 原始咨询记录
   - 私人复盘
   - 临场讲法
   - 未公开素材

## 案例库字段

每个案例建议保留这些字段：

- 标题
- 主题
- 问题背景
- 抽到的牌
- 解读过程
- 反馈结果
- 教学重点
- 适合学生阶段
- 相关牌名
- 相关主题
- 权限级别
- 是否已匿名化

## 权限设计

第一版建议先轻量实现：

- 公开内容：不需要密码
- 团课内容：团课密码或专属链接
- 一对一内容：一对一密码或专属链接
- 私人内容：本地保存，不部署到公开网站

以后可以升级为：

- 学生账号登录
- 不同课程包对应不同资料
- 团课有效期
- 一对一永久权限
- 学生学习记录和收藏

## 技术路线

第一版：

- 静态网站
- 本地资料文件夹
- Markdown 内容
- 简单搜索和筛选
- GitHub Pages / Cloudflare Pages / Vercel 部署

第二版：

- 内容管理后台
- 更稳的权限系统
- 学生账号
- 案例批量导入

第三版：

- 基于你资料的 AI 助教
- 自动生成课堂提纲
- 自动从案例生成练习题
- 自动打标签和归档

## 接下来怎么做

1. 你导出一份幕布资料给我
2. 我先整理出一版目录和标签
3. 你判断哪些内容能给谁看
4. 我把内容放进知识库原型
5. 我们再决定是否部署上线
